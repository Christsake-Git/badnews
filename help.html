Help - Bad News Tracker

API Endpoints

  • /vendors (GET): Returns a list of vendors.
  • /manage_vendors (POST): Add or remove a vendor.
    • JSON format: 
      {
    "action": "add",   // or "delete"
    "vendor_name": "Vendor Name"
}
  • /badnews (GET): Displays the bad news results for each vendor.
  • /force_scan (POST): Manually triggers a new search for bad news for all vendors.

Usage Instructions

This application allows you to manage a list of vendors and track bad news related to them. You can add or remove vendors using the /manage_vendors endpoint. The application will perform a Google search for each vendor to find any recent bad news related to cyber attacks, data breaches, fraud, or scams.

How to Use

  1. To view the list of vendors, navigate to /vendors.
  2. To add a vendor, send a POST request to /manage_vendors with the appropriate JSON payload.
  3. To view bad news, navigate to /badnews.
  4. To force a new scan for bad news, send a POST request to /force_scan.

Notes

  • The application uses Google search to find bad news, so it may be subject to rate limiting.
  • If you encounter a "Too Many Requests" error, the application will automatically retry the search with exponential backoff.

For further assistance, please contact the application administrator.
